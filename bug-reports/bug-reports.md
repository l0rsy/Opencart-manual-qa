# Bug Reports — OpenCart Demo

## BUG-001: Регистрация с односимвольным именем проходит успешно

| Поле | Значение |
|------|----------|
| ID | BUG-001 |
| Severity | Major |
| Priority | Medium |
| Статус | Open |
| Модуль | Auth |
| Тест-кейс | TC-021 |

**Окружение:**
ОС: Windows 10 | Браузер: Chrome | URL: https://demo.opencart.com/index.php?route=account/register

**Шаги воспроизведения:**
1. Открыть страницу регистрации
2. Заполнить First Name: `A`
3. Заполнить остальные поля валидными данными
4. Активировать тумблер Privacy Policy
5. Нажать **Continue**

**Ожидаемый результат:**
Отображается ошибка валидации. Регистрация не выполнена.

**Фактический результат:**
Регистрация выполнена успешно. Отображается "Your Account Has Been Created!"

**Скриншот:** bug-001.jpg

## BUG-004: Некорректное отображение UI корзины после ввода нулевого или нечислового количества товара

| Поле | Значение |
|------|----------|
| ID | BUG-004 |
| Severity | Major |
| Priority | Medium |
| Статус | Open |
| Модуль | Cart |
| Тест-кейс | TC-024, TC-025, TC-026 |

**Окружение:**
ОС: Windows 10 | Браузер: Chrome | URL: https://demo.opencart.com/index.php?route=checkout/cart

**Шаги воспроизведения:**
1. Добавить любой товар в корзину
2. Открыть страницу корзины
3. В поле Quantity ввести `0` (либо `-1`, либо `abc`)
4. Нажать **Update**

**Ожидаемый результат:**
Отображается ошибка валидации либо товар корректно удаляется.
UI корзины отображается корректно.

**Фактический результат:**
Товар удаляется из корзины, однако UI остаётся в некорректном состоянии:
счётчик в шапке не обновляется, отображается вес корзины и пустые строки таблицы.
Состояние корректно восстанавливается только после обновления страницы.

**Скриншот:** bug-004.jpg

## BUG-005: Превышение доступного количества товара обнаруживается только на этапе Checkout

| Поле | Значение |
|------|----------|
| ID | BUG-005 |
| Severity | Minor |
| Priority | Low |
| Статус | Open |
| Модуль | Cart |
| Тест-кейс | TC-027 |

**Окружение:**
ОС: Windows 10 | Браузер: Chrome | URL: https://demo.opencart.com/index.php?route=checkout/cart

**Шаги воспроизведения:**
1. Добавить любой товар в корзину
2. Открыть страницу корзины
3. В поле Quantity ввести `99999`
4. Нажать **Update**
5. Нажать **Checkout**

**Ожидаемый результат:**
Ошибка о недостаточном количестве товара отображается на шаге обновления количества в корзине.

**Фактический результат:**
Количество `99999` принимается без ошибки.
Ошибка отображается только при попытке перейти к оформлению заказа.

**Скриншот:** bug-005.jpg
